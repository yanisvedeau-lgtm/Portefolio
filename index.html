Projet

Vision

Ce projet présente une page vitrine orientée qualité, sans artifices : une structure sémantique complète, des sections cohérentes, et des contenus utiles pour un dépôt open-source.

Fonctionnalités principales

  • Navigation par ancres : accès rapide aux sections.
  • Tableau d’indicateurs : suivi simple et lisible.
  • Formulaire de contact : champs complets, libellés associés.
  • FAQ : détails repliables pour limiter la longueur.

Documentation

Utilisation

Cette page est conçue pour être publiée telle quelle sur GitHub Pages.

Exemple de structure recommandée

/
  index.html
  README.md
  (optionnel) assets/

Si vous ajoutez des ressources (images), privilégiez un dossier assets/ et décrivez-le dans le README.md du dépôt.

Contribuer

Avant de proposer une amélioration, voici une checklist courte et utile.

Checklist

  • But clair : pourquoi ce changement ?
  • Impact : sur le contenu, la structure, l’accessibilité.
  • Lisibilité : titres cohérents et balises sémantiques appropriées.

Un bon changement est celui qui améliore le projet sans augmenter inutilement la complexité.

Pourquoi uniquement du HTML ?

Pour démontrer une base sémantique solide, indépendante de toute couche de présentation ou d’interactivité. C’est un excellent point de départ pour un dépôt public : lisible, stable et facile à auditer.

Comment ajouter un style plus tard ?

Vous pouvez ensuite ajouter un fichier CSS séparé et lier une feuille de style dans le <head>. Ici, le choix volontaire est de rester sans style pour respecter les contraintes.
